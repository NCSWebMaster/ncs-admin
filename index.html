🔒 Super Admin Security Protocol

Account Protection

  • Cannot be deleted via normal user management screens
  • Cannot be demoted through frontend interfaces
  • Server-side authorization required for all changes
  • Final Super Admin protected — system prevents removal of last Super Admin

Removal/Demotion Requirements

  1. Administrator must re-authenticate
  2. MFA verification required (TOTP or email code)
  3. Server-side confirmation secret must be validated (stored in env vars only)
  4. Final confirmation dialog must be completed

Confirmation Secret Protection

  • 🚫 Never exposed to frontend
  • 🚫 Never in React/JavaScript bundles
  • 🚫 Never in API responses or database visible to users
  • ✅ Stored only in secure server environment variables

✓ Audit Logging (Immutable)

Every Super Admin action is logged and cannot be edited:

  • User performing the action
  • User being modified
  • Action type (delete, demote, restore)
  • Timestamp with timezone
  • IP address (if available)
  • Device/browser info (if available)
  • Reason entered by administrator
